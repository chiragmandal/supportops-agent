# Security & Compliance (Fintech SaaS)

## Sensitive data
- Never request full ID card number or full bank account details.
- If needed: request last 4 digits only.

## Suspicious activity / AML
- Any AML/chargeback/fraud suspicion must be escalated to Security immediately.
- Do not share internal detection logic with customers.

## Access requests
- Admin access changes require verification.
- SOC2/ISO requests: provide standard security documentation package via the trust portal process (escalate).
